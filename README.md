# 位置查岗插件 (Location Guard) v2.0.2

当你离家时，手机自动上报信息，机器人会结合聊天记录判断你是否提前报备过出门，读取当前人设和最近对话上下文，并用符合角色风格的语气发送查岗或关心消息。回家时也会自动欢迎你。支持天气感知，下雨会提醒你带伞。

## 功能特性

- 支持WiFi断开检测离家（推荐，无需位置权限，熄屏可用）和GPS地理围栏两种方案
- 通过iPhone快捷指令静默上报，无需手动操作
- 离家和到家均支持天气感知，获取实时坐标、天气和温度数据
- 内置哨兵模型，自动分析最近聊天记录区分"已报备"和"未报备"状态
- 自动读取AstrBot当前人设（全量），生成的消息完全符合你设定的角色风格
- 自动检测UMO，无需手动填写平台名称，支持/new新建对话后自动适配
- 携带最近对话上下文（轮数可配置），让AI的回复更自然连贯
- 三种场景独立提示词（未报备离家/已报备离家/到家），均可在WebUI自定义
- 支持分段回复，按换行自动切割为多条消息，模拟真人打字节奏
- 自动过滤thinking标签和markdown格式符号，确保消息干净自然
- 查岗和到家记录自动写入聊天历史，主模型可感知你的出行状态
- 哨兵模型和查岗模型均支持WebUI下拉选择提供商
- 所有参数均可在WebUI面板配置，无需改代码

## 配置说明

| 配置项 | 说明 |
|--------|------|
| home_lat | 家的纬度坐标 |
| home_lng | 家的经度坐标 |
| alert_distance | 触发查岗的距离阈值，默认500米（仅GPS方案使用） |
| http_port | HTTP监听端口，默认8090 |
| notify_qq | 接收查岗消息的QQ号 |
| bot_qq | 机器人QQ号 |
| bot_platform | 机器人平台名称，一般无需填写，插件会自动检测 |
| onebot_url | NapCat的OneBot HTTP接口地址，默认 http://127.0.0.1:3000 |
| sentinel_provider | 哨兵模型，用于判断是否报备，建议选用轻量模型（支持下拉选择） |
| guard_provider | 查岗消息生成使用的模型，建议选择独立的提供商实例（支持下拉选择） |
| alert_prompt | 未报备离家时的提示词，支持占位符 |
| safe_travel_prompt | 已报备离家时的提示词，支持占位符 |
| home_prompt | 到家时的提示词，支持占位符 |
| enable_segment | 是否启用分段回复 |
| segment_regex | 分段正则表达式（启用分段时默认按换行切割） |
| context_rounds | 生成消息时携带的最近对话轮数，默认6轮 |

提示词支持的占位符：{{current_time}} {{weather}} {{temperature}} {{distance}}

注意：经纬度建议直接修改配置文件填写，WebUI保存时浮点数偶尔会被重置为0。配置文件路径：AstrBot/data/config/astrbot_plugin_location_guard_config.json

## iPhone快捷指令配置

### 方案一：WiFi断开检测（推荐，无需位置权限，熄屏可用）

#### 离家自动化

1. 打开快捷指令 → 自动化 → 新建个人自动化
2. 触发条件选择"Wi-Fi" → 选择你家的WiFi网络（多个WiFi全部选上） → 选择"断开连接时"
3. 运行方式选择"立即运行"
4. 添加操作步骤：
   - 第一步：搜索添加"获取当前天气"（通过天气服务获取实时坐标，绕过位置权限限制）
   - 第二步：搜索添加"获取URL内容"
5. 配置POST请求：
   - URL：http://你的服务器IP:8090/location
   - 方法：POST
   - 请求体：JSON
   - 字段：
     - lat → 天气输出的纬度变量
     - lng → 天气输出的经度变量
     - event → 文本，填 left_home
     - weather → 天气输出的天气状况变量
     - temperature → 天气输出的气温变量
6. 关闭"运行前询问"和"运行时通知"

#### 到家自动化

1. 新建自动化，触发条件选"Wi-Fi" → 选择家庭WiFi → 选"已加入"
2. 运行方式选择"立即运行"
3. 添加操作步骤：
   - 第一步：搜索添加"获取当前天气"
   - 第二步：搜索添加"获取URL内容"
4. 配置POST请求：
   - URL：http://你的服务器IP:8090/location
   - 方法：POST
   - 请求体：JSON
   - 字段：
     - lat → 天气输出的纬度变量
     - lng → 天气输出的经度变量
     - event → 文本，填 arrived_home
     - weather → 天气输出的天气状况变量
     - temperature → 天气输出的气温变量
5. 关闭"运行前询问"和"运行时通知"

### 方案二：GPS地理围栏

1. 打开快捷指令 → 自动化 → 新建个人自动化
2. 触发条件选择"离开" → 在地图上选择家的位置
3. 添加"获取URL内容"，POST请求body包含lat、lng（当前位置变量）和event（left_home）
4. 关闭"运行前询问"和"运行时通知"

注意：GPS方案需要位置权限设为"始终"，部分iOS版本不支持，建议使用方案一

### 如何获取家的坐标

1. 新建一个快捷指令（非自动化）
2. 添加"获取当前位置"步骤
3. 添加"显示提醒"步骤，内容放入"纬度"和"经度"变量
4. 在家中运行该指令，记下弹窗中的数值填入插件配置

## 指令

| 指令 | 说明 |
|------|------|
| /check_status | 查看最近一次查岗记录 |
| /location_report 纬度 经度 | 手动上报位置，用于测试 |
| /get_umo | 获取当前会话的UMO标识 |
| /test_config | 查看当前插件配置 |
| /test_persona | 查看当前人设读取状态 |
| /test_write | 测试聊天历史写入功能 |

## 工作原理

1. iPhone检测到断开家庭WiFi → 通过天气服务获取坐标和天气 → POST到服务器
2. 服务器收到请求 → 哨兵模型读取最近聊天记录判断是否报备
3. 读取AstrBot当前人设（全量） + 最近N轮对话上下文 → 通过独立provider调用AI
4. AI生成符合角色风格的查岗/关心消息 → 按换行分段 → 通过NapCat逐条推送到QQ
5. 消息内容写入对话历史数据库 → 主模型在后续聊天中可感知出行状态
6. 回家连上WiFi → 触发到家自动化 → AI生成欢迎回家消息（同样携带天气数据）

## 项目结构

```
astrbot_plugin_location_guard/
├── main.py                # 主入口，注册插件、初始化配置、调度各模块
├── _conf_schema.json      # 配置定义，WebUI面板自动渲染
├── metadata.yaml          # 插件元信息（名称、版本、作者）
├── README.md              # 使用文档
└── core/                  # 核心功能模块
    ├── __init__.py         # 模块标识
    ├── utils.py            # 工具函数（提示词填充、文本清洗、路径查找等）
    ├── http_server.py      # HTTP服务器，接收iPhone快捷指令的POST请求

    ├── history.py          # 聊天历史读写、UMO自动检测
    └── sentinel.py         # 哨兵模型，判断用户是否报备过出门
```

## 注意事项

- 服务器防火墙需放行8090端口（或自定义端口）
- NapCat需开启HTTP服务（默认端口3000）
- 哨兵模型建议使用gemini-flash等轻量模型以节省成本
- 建议为guard_provider选择一个与日常聊天不同的提供商实例，避免共用同一实例导致限流。可以使用相同的API Key和模型，只要在AstrBot中是不同的提供商即可
- 经纬度请勿在WebUI中修改，直接编辑配置文件更可靠
- 天气服务获取的坐标精度约1公里，不影响查岗功能
- 如果AI生成的消息语气不符合预期，可在WebUI调整提示词或减少context_rounds
- 使用thinking模型（如claude-thinking）时，插件会自动过滤thinking标签，无需额外处理
